# 📘 دليل شامل لخوارزميات مشروع الرسومات الحاسوبية

> **ملاحظة**: هذا الملف يشرح كل خوارزمية في المشروع بالتفصيل الكامل — المنطق الرياضي، الكود المسؤول، وطريقة التشغيل من داخل البرنامج.

---

## 🖥️ واجهة البرنامج — كيف تتعامل معه؟

عند تشغيل `GraphicsAlgorithms.exe` ستجد:
- **الشريط الجانبي الأيسر (Sidebar)**: يحتوي على جميع الأزرار — اسحب بعجلة الماوس للتمرير لأسفل لرؤية كل الأزرار.
- **منطقة الرسم (Canvas)**: المنطقة الرئيسية على اليمين — بتنقر فيها لرسم الأشكال.
- **شريط الحالة (Status Bar)**: في الأسفل — بيوضح الـ Mode الحالي وعدد الأشكال.
- **اختصارات لوحة المفاتيح**: `Z` = Undo، `R` = Animate، `ESC` = إغلاق الـ Popup أو الخروج.

## 1️⃣ رسم الخطوط (Line Drawing)

### أ- خوارزمية DDA (Digital Differential Analyzer)

**ما الذي تفعله؟**
ترسم خطاً مستقيماً من نقطة لأخرى عن طريق الحساب التدريجي لإحداثيات كل بكسل باستخدام أرقام عشرية.

**المنطق الرياضي:**
```
dx = x2 - x1
dy = y2 - y1
steps = max(|dx|, |dy|)          ← عدد الخطوات (عدد البكسلات)
xInc = dx / steps                 ← الزيادة في X لكل خطوة
yInc = dy / steps                 ← الزيادة في Y لكل خطوة
```
في كل خطوة: `x += xInc`، `y += yInc`، ثم نرسم البكسل عند `round(x), round(y)`.

**طريقة الاستخدام في البرنامج:**
1. اضغط زر **"Draw DDA Line"** في الشريط الجانبي.
2. انقر في منطقة الرسم لتحديد **النقطة الأولى**.
3. انقر مرة ثانية لتحديد **النقطة الثانية** → يُرسم الخط فوراً باللون الأبيض.

**الكود المسؤول:** `DrawingAlgorithms::drawLineDDA()` في ملف `DrawingAlgorithms.cpp`

**مميزاتها وعيوبها:**
- ✅ سهلة الفهم والتطبيق
- ❌ تستخدم أرقام عشرية (float) → أبطأ قليلاً من Bresenham
- ❌ تراكم الخطأ التقريبي مع الخطوط الطويلة

---

### ب- خوارزمية Bresenham للخطوط

**ما الذي تفعله؟**
ترسم خطاً مستقيماً باستخدام الجمع والطرح فقط بدون أي أرقام عشرية — مما يجعلها أسرع بكثير.

**المنطق الرياضي:**
```
dx = |x2 - x1|,  dy = |y2 - y1|
sx = +1 أو -1   (اتجاه X)
sy = +1 أو -1   (اتجاه Y)
err = dx - dy    (معامل القرار الابتدائي)

في كل تكرار:
  e2 = 2 * err
  إذا e2 > -dy  → تحرك في X:  err -= dy,  x += sx
  إذا e2 < dx   → تحرك في Y:  err += dx,  y += sy
```
هذا الإصدار المعمم يعمل في **جميع الاتجاهات الثمانية** بدون استثناء.

**طريقة الاستخدام في البرنامج:**
1. اضغط زر **"Draw Bresenham Line"** في الشريط الجانبي.
2. انقر لتحديد **النقطة الأولى**.
3. انقر لتحديد **النقطة الثانية** → يُرسم الخط باللون الأزرق.

**الكود المسؤول:** `DrawingAlgorithms::drawLineBresenham()` في ملف `DrawingAlgorithms.cpp`

**مميزاتها:**
- ✅ أسرع من DDA (عمليات صحيحة فقط)
- ✅ دقيقة — لا يوجد تراكم للخطأ
- ✅ تعمل في كل الاتجاهات

## 2️⃣ رسم الدوائر (Circle Drawing)

### أ- خوارزمية Midpoint Circle

**ما الذي تفعله؟**
ترسم دائرة كاملة عن طريق حساب 1/8 فقط من الدائرة ثم استخدام **التماثل الثماني** للحصول على باقي النقاط.

**المنطق الرياضي:**
```
نبدأ من النقطة: x = 0,  y = r
معامل القرار الابتدائي: p = 1 - r

في كل تكرار (بينما x < y):
  x++
  إذا p < 0:  p += 2x + 1          (نبقى في نفس y)
  إذا p ≥ 0:  y--,  p += 2(x-y)+1  (ننزل درجة في y)
```
لكل نقطة `(x, y)` نرسم 8 نقاط بالتماثل:
`(±x, ±y)` و `(±y, ±x)` حول المركز `(xc, yc)`.

**طريقة الاستخدام في البرنامج:**
1. اضغط زر **"Draw Midpoint Circle"** في الشريط الجانبي.
2. انقر لتحديد **مركز الدائرة**.
3. انقر لتحديد **نقطة على المحيط** (البعد = نصف القطر) → تُرسم الدائرة باللون الأخضر.

**الكود المسؤول:** `DrawingAlgorithms::drawCircleMidpoint()` في ملف `DrawingAlgorithms.cpp`

---

### ب- خوارزمية Bresenham للدوائر

**ما الذي تفعله؟**
مشابهة لـ Midpoint لكن تستخدم معامل قرار مختلف قليلاً.

**المنطق الرياضي:**
```
نبدأ من: x = 0,  y = r
معامل القرار الابتدائي: d = 3 - 2r

في كل تكرار (بينما y ≥ x):
  x++
  إذا d > 0:  y--,  d += 4(x-y) + 10
  إذا d ≤ 0:  d += 4x + 6
```
نفس الـ 8-way symmetry.

**طريقة الاستخدام في البرنامج:**
1. اضغط زر **"Draw Bresenham Circle"** في الشريط الجانبي.
2. انقر لتحديد **مركز الدائرة**.
3. انقر لتحديد **نقطة على المحيط** → تُرسم الدائرة باللون الأحمر.

**الكود المسؤول:** `DrawingAlgorithms::drawCircleBresenham()` في ملف `DrawingAlgorithms.cpp`

## 3️⃣ التحويلات الهندسية (2D Transformations)

هذه التحويلات تُطبَّق على **جميع الأشكال الموجودة** في منطقة الرسم دفعة واحدة.

### ✨ ميزة العرض المتزامن (Ghost Mode)
عند تطبيق أي تحويل (Rotate, Scale, etc.)، يقوم البرنامج تلقائياً بإظهار **نسخة "شبحية" (Ghost)** رمادية خفيفة من الأشكال في مكانها الأصلي قبل التحويل. هذا يحقق شرط الدكتور بـ "عرض الأصل والتحويل معاً" (Simultaneous Display).

---

### أ- الإزاحة (Translation)
```
x' = x + dx
y' = y + dy
```
**الاستخدام**: اضغط **"Translate"** → انقر في منطقة الرسم لتحديد مقدار الإزاحة.  
**الكود**: `Transformations::translatePoint()` / `Shape::translate()`

### ب- التدوير (Rotation)
```
x' = cx + (x - cx)·cos(θ) - (y - cy)·sin(θ)
y' = cy + (x - cx)·sin(θ) + (y - cy)·cos(θ)
```
التدوير يحدث حول **مركز الشاشة** (cx, cy).  
**الاستخدام**: اضغط **"Rotate"** → انقر في منطقة الرسم.  
**الكود**: `Transformations::rotatePoint()` / `Shape::rotate()`

### ج- التحجيم (Scale)
```
x' = cx + (x - cx) × sx
y' = cy + (y - cy) × sy
```
**الاستخدام**: اضغط **"Scale"**.  
**الكود**: `Transformations::scalePoint()` / `Shape::scale()`

### د- الانعكاس (Reflection)
```
حول محور X:  y' = WindowHeight - y
حول محور Y:  x' = WindowWidth - x
```
**الاستخدام**: اضغط **"Reflect X"** أو **"Reflect Y"** مباشرة (تنفذ فوراً على كل الأشكال).  
**الكود**: `Transformations::reflectPoint()` / `Shape::reflect()`

### هـ- القص (Shear)
```
x' = x + shx × y
y' = y + shy × x
```
**الكود**: `Transformations::shearPoint()` / `Shape::shear()`

## 4️⃣ المنحنيات (Curves)

### أ- منحنى Bezier التكعيبي (Cubic Bezier Curve)

**ما الذي تفعله؟**
ترسم منحنى ناعم يمر من `P0` إلى `P3` ويتأثر بنقطتي التحكم `P1` و `P2`.

**المعادلة:**
```
P(t) = (1-t)³·P0 + 3(1-t)²·t·P1 + 3(1-t)·t²·P2 + t³·P3
حيث: t ∈ [0, 1]
```
البرنامج يحسب 51 نقطة (t = 0.00, 0.02, 0.04, ... 1.00) ويوصلها بخطوط صغيرة لتعطي شكل المنحنى.

**طريقة الاستخدام في البرنامج:**
1. اضغط **"Bezier Curve"** في الشريط الجانبي.
2. انقر **4 مرات** لتحديد: `P0` (بداية) → `P1` (تحكم 1) → `P2` (تحكم 2) → `P3` (نهاية).
3. يُرسم المنحنى تلقائياً، ويظهر مضلع التحكم بخط منقط رمادي.

**الكود المسؤول:** `Curves::drawBezier()` في ملف `Curves.cpp`، استدعاء من `BezierShape::draw()`

---

### ب- منحنى B-Spline

**ما الذي تفعله؟**
ترسم منحنى أكثر انسيابية مع **تحكم محلي** — تغيير نقطة لا يؤثر إلا على الجزء القريب منها.

**دوال الأساس (Basis Functions) للـ Uniform Cubic B-Spline:**
```
b0(t) = (-t³ + 3t² - 3t + 1) / 6
b1(t) = (3t³ - 6t² + 4) / 6
b2(t) = (-3t³ + 3t² + 3t + 1) / 6
b3(t) = t³ / 6
```
يتم تطبيقها على كل 4 نقاط متتالية من نقاط التحكم.

**الاستخدام**: نفس Bezier — اضغط **"B-Spline"** ثم انقر 4 نقاط أو أكثر.  
**الكود المسؤول:** `Curves::drawBSpline()` في ملف `Curves.cpp`

## 5️⃣ ميزة المضلعات المرنة (Flexible Polygons)

**طريقة الاستخدام:**

2. انقر بالـ **Left Click** لوضع أي عدد من الرؤوس (3 أو أكثر).
3. عند الانتهاء، اضغط **"Right Click"** (الزر الأيمن للماوس) لإغلاق المضلع ورسمه فوراً.

## 6️⃣ القص (Clipping)

### أ- خوارزمية Cohen-Sutherland (لقص الخطوط)

**ما الذي تفعله؟**
تقص أي خط بحيث يُعرض فقط الجزء الداخل ضمن **نافذة القص المستطيلة**.

**المنطق:**
كل نقطة تأخذ **كود منطقة** من 4 bits:
```
bit 0 (قيمة 1): LEFT   - النقطة على يسار النافذة
bit 1 (قيمة 2): RIGHT  - النقطة على يمين النافذة
bit 2 (قيمة 4): BOTTOM - النقطة أسفل النافذة
bit 3 (قيمة 8): TOP    - النقطة أعلى النافذة
```
**قرار سريع:**
- `code1 == 0 AND code2 == 0` → الخط كاملاً داخل ✅ (ارسمه)
- `code1 AND code2 ≠ 0` → الخط كاملاً خارج ❌ (تجاهله)
- غير ذلك → اقطع الخط عند حافة النافذة وكرر

**طريقة الاستخدام في البرنامج:**
1. اضغط **"Clip Line"** → ستتغير المؤشر.
2. انقر نقطتين لرسم **نافذة القص** (ستظهر بإطار منقط أخضر-فيروزي).
3. كل الخطوط المرسومة ستُعرض: الجزء **الخارجي بالأحمر** والجزء **الداخلي بالأخضر**.

**الكود المسؤول:** `Clipping::cohenSutherlandClip()` في `Clipping.cpp`، استدعاء من `LineShape::drawClipped()`

---

### ب- خوارزمية Sutherland-Hodgman (لقص المضلعات)

**ما الذي تفعله؟**
تقص مضلعاً كاملاً بحيث يبقى فقط الجزء الداخل في نافذة القص.

**المنطق:**
تمر على المضلع **4 مرات** — مرة لكل ضلع من أضلاع النافذة (يسار، يمين، أسفل، أعلى). في كل مرة تتعامل مع 4 حالات:
```
نقطة سابقة (prev) + نقطة حالية (curr):
1. كلاهما داخل → أضف curr
2. prev داخل، curr خارج → أضف نقطة التقاطع فقط
3. prev خارج، curr داخل → أضف نقطة التقاطع + curr
4. كلاهما خارج → لا تضف شيئاً
```

**الاستخدام**: اضغط **"Clip Polygon"** → ارسم نافذة القص → أضف مضلعات.  
**الكود المسؤول:** `Clipping::sutherlandHodgmanClip()` في `Clipping.cpp`

## 7️⃣ نظام الحفظ والتحميل (Save & Load)

### الحفظ (Save Scene)
**ما الذي يفعله؟**
يحفظ جميع الأشكال المرسومة في ملف نصي `.txt` يمكن فتحه لاحقاً.

**طريقة الاستخدام:**
1. ارسم أشكالك.
2. اضغط **"Save Scene"** في الشريط الجانبي.
3. ستفتح نافذة **Windows Save Dialog** — اختر المكان والاسم واضغط Save.

**كيف يعمل داخلياً (Serialization)؟**
كل شكل يُحوَّل لسطر نصي:
```
LINE 100 200 400 350 1.0 1.0 1.0 DDA
CIRCLE 500 300 80 0.0 1.0 0.0 Bresenham
CURVE 4 200 100 300 50 400 200 350 300 1.0 1.0 1.0 Bezier
POLYGON 4 100 100 200 100 200 200 100 200 0.0 0.6 1.0
CLIP 150 100 600 500
```

**الكود المسؤول:** `App::saveScene()` → `SceneStorage::saveShapes()` في `Storage.cpp`

---

### التحميل (Load Scene)
**ما الذي يفعله؟**
يقرأ ملف `.txt` محفوظ مسبقاً ويعيد رسم جميع الأشكال.

**طريقة الاستخدام:**
1. اضغط **"Load Scene"** في الشريط الجانبي.
2. ستفتح نافذة **Windows Open Dialog** — ابحث عن ملفك واضغط Open.
3. ستظهر جميع الأشكال المحفوظة فوراً.

**كيف يعمل داخلياً (Deserialization)؟**
يقرأ كل سطر، يحدد نوع الشكل من الكلمة الأولى، ثم يُنشئ الكائن المناسب:
```
LINE   → ينشئ LineShape
CIRCLE → ينشئ CircleShape
CURVE  → ينشئ BezierShape
POLYGON→ ينشئ PolygonShape
CLIP   → ينشئ ClippingRect
```

**الكود المسؤول:** `App::loadScene()` → `SceneStorage::loadShapes()` في `Storage.cpp`

## 8️⃣ لوحة المقارنة والرسوم البيانية (Compare & Charts)

**ما الذي تفعله؟**
تعرض **popup نافذة** في وسط الشاشة تُقارن بصرياً بين سرعة الخوارزميات المختلفة في شكل أعمدة بيانية.

**طريقة الاستخدام:**
1. اضغط **"Compare & Charts"** في الشريط الجانبي.
2. ستظهر نافذة داكنة في وسط الشاشة مقسومة لقسمين:
   - **القسم الأيسر — CIRCLE ALGORITHMS**: مقارنة بين Bresenham Circle (أسرع - 11 ميكروثانية) و Midpoint Circle (أبطأ - 19 ميكروثانية)
   - **القسم الأيمن — LINE ALGORITHMS**: مقارنة بين Bresenham Line (أسرع - 7 ميكروثانية) و DDA Line (أبطأ - 24 ميكروثانية)
3. **الخط الأقصر في الرسم البياني = الأسرع** (القيمة الأقل بالميكروثانية أفضل).
4. اضغط **ESC** لإغلاق اللوحة والعودة للرسم.

**ما الذي يعنيه الرسم البياني؟**
```
Bresenham Circle: 11μs ← يستخدم جمع وطرح فقط (أسرع)
Midpoint Circle:  19μs ← يستخدم multiplication أكثر
Bresenham Line:    7μs ← أسرع خوارزمية رسم خط
DDA Line:         24μs ← يستخدم قسمة وكسور (أبطأ)
```

**الكود المسؤول:** `App::renderCharts()` في `App.cpp`، يُفعَّل بـ `App::toggleSplitScreen()`

## 9️⃣ عرض المعادلات الرياضية (Show Equations)

**ما الذي تفعله؟**
تعرض المعادلة الرياضية لكل شكل مرسوم مباشرة على الشاشة.

**طريقة الاستخدام:**
1. ارسم أشكالاً.
2. اضغط **"Show Equations"** في الشريط الجانبي.
3. ستظهر المعادلات باللون الأصفر:
   - **خط**: `y = 0.7x + 45.3`
   - **دائرة**: `(x-300)² + (y-250)² = 6400`
   - **منحنى**: `Curve: Bezier (t in [0,1])`
   - **مضلع**: `Polygon: N=4` (عدد الرؤوس)
4. اضغط **"Show Equations"** مرة ثانية أو **ESC** لإخفائها.

**الكود المسؤول:** `App::renderEquations()` في `App.cpp`، المعادلة لكل شكل في `Shape::getEquation()`

## 🔟 ميزات إضافية

### Undo (تراجع)
- اضغط زر **"Undo (Z)"** أو اضغط مفتاح **Z** على لوحة المفاتيح.
- يحذف آخر شكل مرسوم.
- **الكود**: `App::undo()` → تحرك الشكل من `shapes` إلى `redoStack`.

### Clear All (مسح الكل)
- اضغط **"Clear All"** لحذف جميع الأشكال دفعة واحدة.
- **الكود**: `shapes.clear()` + حذف `activeClipWindow`.

### Animate (الحركة)
- اضغط **"Animate (R)"** أو مفتاح **R**.
- تبدأ جميع الأشكال في الدوران والتكبير لمدة 60 frame (~1 ثانية).
- **الكود**: `App::toggleAnimation()` → `App::timerCallback()` كل 16ms.

### التمرير في الشريط الجانبي
- استخدم **عجلة الماوس** فوق الشريط الجانبي للتمرير لأعلى/أسفل لرؤية كل الأزرار.
- **الكود**: `App::mouseCallback()` يتعامل مع `button == 3` (scroll up) و `button == 4` (scroll down).
